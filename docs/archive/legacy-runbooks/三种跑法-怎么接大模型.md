# 三种跑法 · 提示词怎么接大模型

> 归档说明：这是历史外部 LLM 接法说明。当前无 agent 跑法以 `docs/MVP-ARCHITECTURE-HANDOFF.md` 的“没有 agent 时，怎么用外部 LLM 跑完整流程”为准。

> 背景：本系统的脚本只「出题」（`make-prompt.py` 生成 `.txt` 提示词），不「答题」。
> 中间那个读提示词、产出 JSON 的大模型靠你「自带」。下面三种自带方式，按需要选。
> 不管哪种，最后都由 `ingest.py` 收口安装（禁词/5步/forced-hints 硬门照旧兜底），成品只进 `data/today`。
> 完整三层流程见 `prompts/每日提示词总表.md`；本文件只讲「答题那一环怎么接」。

以教育赛道 `acct-xiaozhu-edu-xhs` 为例，`D` = 当天日期（如 2026-06-24）。

---

## 方案① Agent 当场充当大模型（现在就能用，零改造）★日常推荐

**适用**：手边有能读写文件的 agent（Cowork 里的 Claude / Claude Code / Codex）。agent 自己就是那个「外部 LLM」。

**怎么用**：一句话指派给 agent——

> 「给教育号 acct-xiaozhu-edu-xhs 跑 2026-06-24 的今日推荐，整条跑通。」

**agent 照这个协议做**（它内部就是在替你答题）：

1. 层1 抓取：联网搜**真实**热点（缺料不捏造）→ 按 `prompts/hotspot-broad.md` 产出公共池候选、按 `prompts/tracks/education-yowow/search.md` 产出赛道候选 → 每条过 `prompts/neutralize.md` 中立化 → 写 `data/hotspots/$D.json` 和 `data/hotspots/tracks/education-yowow/$D.json`。
2. `python3 scripts/status.py --date $D` 自检池子。
3. 层2：`make-prompt.py ... --step match` → agent 逐条读 `prompts/match-*.txt`、**当场作答** tier → 写 `_inbox/match-*.json`。
4. 层3：`make-prompt.py ... --step generate`（只对非 skip）→ agent 对照 `prompts/tracks/education-yowow/analysis.md` 七步骨架作答 → 写 `_inbox/generate-*.json`。
5. `ingest.py ... _inbox --date $D` 安装 → `status.py` 自检。
6. 用人话汇报「✅发什么 / 🤔为什么 / 🚫别蹭」。

**优点**：今天就能出内容、判断有质量、零代码改造。
**缺点**：要人喊 agent，不是无人值守。

## 方案② 人工 copy-paste（零依赖，完全可控）

**适用**：没有 agent，或想自己掌控每一步、亲自把关。把提示词贴进 ChatGPT / Claude 网页，把回答粘回文件。

**步骤**：

1. **层1 公共池**：打开 `prompts/hotspot-broad.md`，把正文里的 `{date}` 换成当天，整段贴进网页大模型 → 拿到 JSON 数组 → 对每条再用 `prompts/neutralize.md` 中立化 → 按 `config/today-hotspots.demo.json` 的字段，把结果存成 `data/hotspots/$D.json`（每条 `scope:"broad"`）。
2. **层1 赛道池**：同样用 `prompts/tracks/education-yowow/search.md` → 中立化 → 存 `data/hotspots/tracks/education-yowow/$D.json`（每条 `scope:"track:education-yowow"`）。
3. `python3 scripts/status.py --date $D` 自检。
4. **层2 判断**：`python3 scripts/make-prompt.py acct-xiaozhu-edu-xhs --date $D --step match` → 打开每个 `data/runs/$D/acct-xiaozhu-edu-xhs/prompts/match-<id>.txt`，整段贴进网页 → 把回答 JSON 存成 `_inbox/match-<id>.json`。
5. **层3 分析**：`make-prompt.py ... --step generate` → 打开 `generate-<id>.txt`，**连同 `prompts/tracks/education-yowow/analysis.md` 一起贴**（让模型按七步骨架剖析）→ 回答存 `_inbox/generate-<id>.json`。
6. **安装**：`python3 scripts/ingest.py acct-xiaozhu-edu-xhs data/runs/$D/acct-xiaozhu-edu-xhs/_inbox --date $D` → `status.py` 自检。

**格式红线**（错了 ingest 会拒收）：
- `_inbox` 文件名里的 `<id>` 必须和热点池里的 `hotspot_id` 完全一致。
- JSON 存 UTF-8、不要被编辑器转义中文。
- 非 skip 必须 ≥3 桥、每桥 5 步齐、草稿三件套齐、`external_terms_check:true`。

**优点**：零依赖、每条都过你眼。
**缺点**：慢、易手抖出格式错。

## 方案③ API 自动化（已实现 `scripts/answer.py`，填 key 即用）

**适用**：想无人值守、可定时，让「答题」这步自动跑。

**一次性配置**：在 `yowow-adaptation/.env.local` 写入（已被 .gitignore，不进仓库）：

```bash
ANTHROPIC_API_KEY=sk-ant-...        # 必填
MODEL_NAME=claude-sonnet-4-6        # 可选，默认即此
# ANTHROPIC_BASE_URL=https://api.anthropic.com   # 可选，走代理时改
```

> 注：`answer.py` 读的是进程环境变量。命令行跑前先 `export $(grep -v '^#' .env.local | xargs)` 或手动 `export ANTHROPIC_API_KEY=...`。

**每天怎么跑**（层1 抓热点仍建议人/agent 把关真实性；层2/3 自动）：

```bash
D=2026-06-24

# 层1：建好两个热点池（同方案①/②，真实热点不交给无人值守瞎编）
python3 scripts/status.py --date $D

# 层2 自动答题
python3 scripts/make-prompt.py acct-xiaozhu-edu-xhs --date $D --step match
python3 scripts/answer.py      acct-xiaozhu-edu-xhs --date $D --step match     # ← 自动调 LLM 写 _inbox/match-*.json

# 层3 自动答题（answer.py 会自动把该赛道 analysis.md 作为 system 注入）
python3 scripts/make-prompt.py acct-xiaozhu-edu-xhs --date $D --step generate
python3 scripts/answer.py      acct-xiaozhu-edu-xhs --date $D --step generate  # ← 自动写 _inbox/generate-*.json

# 安装
python3 scripts/ingest.py acct-xiaozhu-edu-xhs data/runs/$D/acct-xiaozhu-edu-xhs/_inbox --date $D
python3 scripts/status.py --date $D
```

**answer.py 做了什么 / 没做什么**：
- 做：读 `prompts/<step>-*.txt` → 调 Anthropic messages API → 抽 JSON（复用 `prompt_loader.extract_json`，能从带前后语的回答里抽出）→ 写同名 `_inbox/*.json`。generate 步自动注入 `prompts/tracks/<track>/analysis.md` 作 system。
- 没做：不发明流程、不绕过 ingest。禁词/5步/forced-hints 仍由 `ingest.py` 硬门兜底——**模型答错也会被拦**。
- 无 key：直接报错退出，不伪造。
- 自测：`python3 scripts/answer.py --selftest`（离线，不需真 key）。

**建议**：
- `strong_pick` 上线前最好人审一眼（API 自动跑也会偶尔过浅或踩争议）。
- 想每天定时：用调度（cron / 定时任务）串 `make-prompt → answer → ingest → status`，并把当天结果发给老板审。
- 层1 抓热点别全自动瞎编：真实性是底线，至少保留人/agent 把关或用可信热榜接口。

**成本意识**：每账号每天 ≈ 热点数 × 2 步（match+generate）次 API 调用，按 token 计费；先小跑估一天的量再决定是否定时。

---

## 三种怎么选

| 场景 | 选 |
|---|---|
| 今天就要出内容、手边有 agent | ① |
| 没 agent、想亲自把关每条 | ② |
| 要每天自动、能接受先配 key + 写点调度 | ③ |

三种产出完全一样（都写进 `_inbox/` 再由 `ingest.py` 安装），可随时混用、互不冲突。
