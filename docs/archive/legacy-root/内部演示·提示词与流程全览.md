# yowow-adaptation 内部演示：提示词与流程全览

> 归档说明：这是早期内部演示文档。当前提示词入口以 `prompts/README.md` 为准，工程流程以 `docs/MVP-ARCHITECTURE-HANDOFF.md` 为准。

> 整理日期：2026-06-12  
> 适用场景：内部同事演示、新成员培训、博士方法论讲解

---

## 一、系统架构一句话

**热点池 → 提示词渲染 → 外部 LLM 回贴 → 硬门收口 → 网站**

每天发生的事：系统把当天热点 × 账号配置拼成提示词，交给外部 LLM 判断"能不能发、怎么发"，LLM 的回贴必须通过 `ingest.py` 硬门才能写进网站数据，坏回贴永远进不了线上。

---

## 二、六个核心概念

| 概念 | 是什么 | 不是什么 |
|---|---|---|
| **赛道 Track** | 方法论对象，唯一事实源在 `config/tracks/<id>.json` | 不是账号，不随平台变 |
| **账号 Account** | 渠道对象，只放业务事实（人设/口吻/平台） | 不保存方法论副本 |
| **公共池** | 无行业偏向的大盘扫描结果 | 不含赛道定向召回 |
| **赛道池** | 按赛道母题定向搜索的结果 | 不入公共池 |
| **内部锚 internal_lens** | 博士判断用的专业术语（如远迁移）| 永不出现在成品里 |
| **对外词 external_vocab** | 把内部锚翻译成受众听懂的人话 | 不是行业黑话 |

---

## 三、每日五步流程

```
步骤1  存热点    data/hotspots/YYYY-MM-DD.json（公共池+赛道池）
步骤2  生成提示词  python3 scripts/make-prompt.py <account_id> --date <D>
步骤3  外部LLM   逐条粘贴 prompts/*.txt → 收回贴放 raw/
步骤4  收口安装   python3 scripts/ingest.py <account_id> raw/ --date <D>
步骤5  刷新网站   git push → Vercel 自动部署
```

## 四、八个提示词模板（原文 + 用途说明）

---

### 4.1 `hotspot-broad.md` · 公共池大盘扫描

**用途**：每天一次，无赛道偏向，扫"普通人在平台上真正自发热议的话题"。  
**谁改**：博士/老板。改扫哪些平台、什么算真热议。  
**铁律**：模板内不得出现任何赛道、行业、产品的字眼。

```
任务：找出过去两周普通人在小红书、抖音等平台上真正自发热议的话题

要的：普通人自己发帖、晒图、吐槽、对线、玩梗的话题（主角是用户，不是记者）
不要：只有官方通报、媒体报道、无UGC痕迹的

扫描面：
1. 生活方式风潮
2. 生活争议与站队
3. 育儿与家庭圈热帖
4. 打工人话题
5. 身体与状态
6. 梗与格式
7. 关系与情感
8. 海外平台正在火的

入池门槛（每条必须全过）：
① 平台原生证据（能给出tag/声量）
② 第一人称浓度（"我家/我自己"的现身说法）
③ 跨人群（≥2类人在聊）
④ 有冲突结构
⑤ 能读出2种以上焦虑/欲望

总量 10-20 条，不做任何行业取舍。
```

**输出格式**：JSON 数组，每条含 title / summary / platform_evidence / conflict_hint / problem_dimensions_hint / est_heat_score_10。

---

### 4.2 `hotspot-search.md` · 赛道定向搜索

**用途**：每个赛道每天一次，按赛道母题召回过去一个月能自然引出该赛道的热点。  
**谁改**：博士。改召回口径（什么算自然引出、什么算硬蹭）。

**关键口径**（博士定版）：
- 只要"自然引出"，不要"硬蹭得上"
- 时间窗：过去一个月
- 每条方向召回 0-3 条，召回不到就空着，不为凑数降低标准

**变量占位**：`{date}` `{track}` `{search_brief}` `{directions}`

**输出格式**：JSON 数组，每条含 title / source_hint / hit_directions / why_natural / est_heat_score_10。

---

### 4.3 `neutralize.md` · 热点中立化

**用途**：把抓取层的原始候选富化成"不绑任何赛道"的中立热点，全赛道共享。  
**谁改**：博士/老板。改怎么抽现象、怎么判情绪、什么算候选问题维度。

**核心口径**：
- 只读出、不搬运（不整段抄原料 summary）
- 中立 = 不预设赛道（phenomenon/情绪/维度都不替任何赛道说话）
- 缺料就老实说缺料，`needs_human=true`，不硬编

**输出格式**：
```json
{
  "needs_human": false,
  "neutral": {
    "phenomenon": "客观发生了什么",
    "spread_emotion": "大家在激动/焦虑/嘲讽什么",
    "candidate_problem_dimensions": ["可能戳中的真实问题1", "问题2"]
  }
}
```

**三个必填字段**：phenomenon / spread_emotion / candidate_problem_dimensions（至少1个，建议≥2个）

---

### 4.4 `hotspot-match.md` · 热点 × 赛道相关性匹配

**用途**：判断一条中立热点跟某条赛道的相关性，分三档：strong_pick / maybe / skip。  
**谁改**：博士/老板。改什么算高匹配、什么算牵强。

**判断两个维度**：
- **相关度**：戳中的真实焦虑和本赛道客户焦虑契合多紧（热点自身流量大小不算）
- **自然度**：用 `internal_lens` 判断——能不能自然承载本赛道主张，还是绕很远才搭得上

**定档规则**：
- 自然度 < 5，或必须硬掰 → **skip**
- 相关度高且自然度高 → **strong_pick**
- 介于两者之间 → **maybe**

**老板批注（最终验收标准）**：看的不是"能不能写出一篇内容"，而是"老板看完会不会觉得：这个热点我原来不会接，但它帮我接得很自然"。需要强行跳跃才能接上的，一律 skip。

**输出格式**：
```json
{
  "tier": "strong_pick | maybe | skip",
  "relevance_score": 0.0,
  "naturalness_score": 0.0,
  "why_relevant": "给老板看的一句人话，零内部术语",
  "skip_reason": "仅当skip：为什么不建议蹭"
}
```

---

### 4.5 `content-generate.md` · 内容方案生成

**用途**：针对"一条热点 × 一条赛道 × 一个平台 × 一个人设"，出一份可发布的内容方案。  
**谁改**：老板/运营改口吻结构；博士改桥梁逻辑方法论。

**自然逻辑链 5 步（每条 bridge_path 的骨架）**：

| 步 | 字段 | 写什么 |
|---|---|---|
| 1 现象 | `phenomenon` | 这条热点出现了什么现象（客观） |
| 2 真实问题 | `real_problem` | 受众什么真实焦虑 |
| 3 赛道关系 | `track_relation` | 这个焦虑和本赛道什么关系（人话，不用内部锚） |
| 4 产品价值 | `product_value_support` | 这条桥梁路径如何支撑产品价值 |
| 5 平台表达 | `platform_expression` | 如何落成适合该平台的表达 |

**规则**：
- 连接牵强 → 直接 skip，绝不硬生成
- 能接，给至少 3 条不同桥梁路径，选最自然的一条落成成品
- 成品带：切入角度 / 强钩子标题 / 开头 / 主体结构 / 人设口吻

**硬门护栏（ingest.py 会自动校验）**：
- 成品零禁词（internal_lens 里的术语、赛道禁词、账号禁词）
- 非 skip 必须 ≥3 条 bridge_paths，每条 5 步字段齐全
- 牵强即 skip，绝不硬生成

---

### 4.6 `daily-summary.md` · 每日人话汇总

**用途**：把 ingest 安装完的 latest.json 翻译成给老板的三段话。  
**谁改**：老板/运营。改口吻、详略。

**固定三段输出**：
```
✅ 今天发什么
   每条 strong_pick：标题 + 一句话钩子（为什么这条今天值得发）。
   没有 strong_pick 就直说「今天没有必发的，别硬发」。

🤔 为什么能发
   每条桥梁逻辑一句人话：热点戳中了什么焦虑 → 和你的生意怎么对上。
   零内部术语、零字段名。

🚫 哪些别蹭
   每条 skip + skip_reason 原话。
```
maybe 单列一行：「可发但建议拍板：…」

---

### 4.7 `bridge-motif.md` · 新赛道桥梁母题起草

**用途**：给新赛道起草"桥梁母题"，写进 `config/tracks/<id>.json`。系统真正的"脑子"入口。  
**谁改**：主要是博士。引擎不发明任何桥梁内容，只把模板交给 LLM 起草，博士抽审定稿。

**起草方法论（博士写的核心原则）**：

1. 先看热点这一端、再看赛道这一端，桥要自然长出来，不是硬凑
2. 区分"内部理解锚"（专业术语，后台用）和"对外表达词"（受众听懂的人话）
3. 禁词 = 把内部锚里的黑话列出来，明令对外不准出现
4. example_bridges 给 2~4 条 few-shot 种子

**输出格式（写进 tracks 文件）**：
```json
{
  "bridge": {
    "internal_lens": "后台判断视角，专业术语可用，永不外显",
    "external_vocab": ["对外人话词1", "词2", "词3"],
    "forbidden_terms": ["对外禁词"]
  },
  "example_bridges": [
    { "hotspot_hint": "什么样的热点", "real_problem": "戳中的真实问题", "track_relation": "和本赛道的关系" }
  ]
}
```

---

### 4.8 `bridge-directions.md` · 热点搜索方向起草

**用途**：给新赛道起草 3-6 条深层搜索母题，写进 tracks 文件的 `search_directions`。  
**谁改**：博士。"什么算深层母题"是本系统赛道差异的源头。

**母题三关检验法**：

1. **落在机理/深层问题层，不是行业泛词**  
   反例：「剃须舒适」  
   正例：「根治须部假性毛囊炎（内生发与红肿的物理机制）」  
   检验：这条母题能解释"什么样的新闻会天然戳中它"吗？解释不了就是泛词。

2. **每条 = 一个可成桥的真实问题**  
   向上接热点：能写出 2-3 类会自然引出它的事件  
   向下接产品：产品价值真的能回应它

3. **每条都能翻译成对外人话**  
   母题可以带专业词（后台搜索用），但必须能写出受众听懂的对应说法

## 五、`ingest.py` 硬门清单（系统自动校验的全部规则）

| 拒收条件 | 说明 |
|---|---|
| JSON 抽不出来 | LLM 回贴格式错误 |
| match 结果缺 `tier` / `skip_reason` | 字段不完整 |
| generate 结果缺 AdaptationOutput 必填字段 | 字段不完整 |
| non-skip 少于 3 条 bridge_paths | 桥梁路径不够 |
| 桥梁 5 步缺字段 | 逻辑链不完整 |
| hotspot_id / track_id / platform_id 不一致 | 账号串了 |
| generate 档位高于 match 档位 | 自动封顶 |
| 成品命中禁词（内部术语/赛道禁词/账号禁词） | **降级 skip，不进可发区** |

## 六、博士判断教义（核心摘录）

### 博士看热点的三步顺序

1. **先问用途**：泛流量？种草？精准流量？用途不同打法完全不同
2. **再问本质**：这件事背后"谁在干什么"？（找到再往上看的那一层）
3. **找超然观点**：不站正方也不站反方，找两个对立观点之上的更深一层结构

### 超然观点公式

**表面争议 → 它暴露了什么客观现实 → 顺着现实指出正确的发展道路**

**例：英国限制儿童和 AI 聊天**  
普通分析：站"AI 危险"或"限制过度"  
博士分析：这事被政府提出来，反过来说明真人社区已成信息茧房——AI 思路反而更开阔。正确道路不是限制，而是让人通过 AI 学会 AI 不能替代的东西。争议大→标 maybe，交博士收口。

**例：高校学生用 AI 写论文**  
普通分析：站"学生堕落"或"工具无罪"  
博士分析：AI 很容易产出结果，说明研究内容本身已过时。首先要变的不是评估系统，是教育内容要变成 AI 原生。骂学生的人没做"升华"这一步。

### 不碰清单（博士明确说不要碰）

1. 政府强力行为 / 监管立法（本质多是寻租）
2. 涉诉讼的事（尤其美国律师团文化）
3. 天灾人祸蹭热点
4. 失业焦虑贩卖（尤其点名中国的）
5. 细节不足的热点（"不知道他想干什么，不足以支撑深度剖析"→直接 skip）
6. 太小众没人关注的

### 争议热点处理法

客观现实带来的大争议**可以做、值得做**（评论热度高）。  
规则：初稿大胆把现实点透，**最后必须有平衡的高级观点收口——这一步是博士的拿手戏**。  
系统映射：标 maybe，risk_note 写"争议大，发布前交博士加平衡收口"。

## 七、真实跑批案例（2026-06-12）

### 教育赛道·小猪号（15 条判定）

**两条必发（strong_pick）**：

**① 屏幕争夺战**（hs-20260612-212，heat 9.5）  
热点：暑假临近，没收/断网/吼叫全失效，家长求规则模板的帖子刷屏  
自然逻辑链：家长在求"禁"的方法 → 真实问题是屏幕赢了十年因为单方面打压供给端 → 本赛道主张：问题在供给质量不在禁令 → 产品落点：暑期的好供给就是屏幕争夺战的解法 → 小红书：《暑假还没开始我先认输？不，今年换打法：不堵了，卷供给》

**② 乐悠懂事视频**（hs-20260612-205，heat 9.0）  
热点：《爸爸当家5》乐悠主动安抚妹妹5000万播放，全网吵"被迫懂事"  
自然逻辑链：5000万播放证明"懂事"戳中了家长集体焦虑 → 真实问题是两种懂事被混淆：消耗型 vs 滋养型 → 本赛道主张：区分两种能力才是正确引导 → 产品落点：滋养型懂事是孩子主动建立结构的能力 → 小红书：《看完乐悠那段5000万播放的视频，别急着心疼"懂事"的孩子》

**演示彩蛋：禁词硬门真实拦截案例**

热点 hs-20260612-214（"抓得紧还是放得开？吵的其实是两个目标函数"）  
初稿：p1 现象句写的是"鸡娃派和快乐派又开吵"  
系统动作：content_gate 检测到 "鸡娃" 命中赛道禁词 → 自动降级 skip，不进可发区  
处理：改写现象句为"『抓得紧』派和『放得开』派又开吵" → 重新入档，升回 maybe  
意义：**系统守住了口径，这条内容进入了拍板区，而不是假装跑过了硬门**

## 八、话术速查

| 说什么 | 系统做什么 |
|---|---|
| 「抓今天的热点」 | 公共池扫描 + 全赛道定向搜索 |
| 「XX号今天发什么」 | 给该账号跑 match + generate + ingest |
| 「所有账号今天发什么」 | 遍历全部 active 账号 |
| 「今天一条龙」 | 抓热点 + 全账号推荐，一次出完 |
| 「新增一个账号：……」 | 判断赛道是否存在，走快速通道或赛道车间 |
| 「给XX赛道重写搜索方向」 | 重跑 bridge-directions 模板，status 回 draft，博士定稿 |
