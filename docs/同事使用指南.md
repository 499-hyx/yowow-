# 内容桥梁台 · 同事使用指南

> 一句话：这个网站每天告诉你——**你的账号今天该发什么、为什么能发、哪些热点别蹭**。
> 你只需要看、复制、打分。不需要懂任何技术。

## 一、每天打开看什么（3 分钟）

1. **打开首页「今日总览」**：最上面是今天的整体情况（多少热点、几条必发/拍板/别蹭）；往下每个账号一行，三个格子直接给结论：今天发什么 / 为什么能发 / 哪些别蹭。
2. **点你的账号**进账号主页：
   - 🟢 **必发**：系统有把握的，直接用。
   - 🟡 **拍板**：能发但角度有点绕或争议大，**你拍板**；标了风险提醒的，发布前自己确认边界。
   - ⚪ **别蹭**：每条都写了人话理由（比如"流量大但跟你的生意关系弱"），看一眼理由就行，别硬蹭。
3. **点开一条内容卡**：完整脚本/文案在里面，**一键复制**就能拿去发。想知道"系统凭什么这么推"，点"打开后台视图"能看到完整的推理过程。

## 二、打分（很重要，1 分钟）

每张内容卡下面有三个问题：**这条能发吗 / 连接自然吗 / 角度对吗**（各 1-5 分）+ 一行想说的话。
请发布前后顺手打一下。你的分数会回流给系统——**它是系统越来越懂你的号的唯一途径**。打分不是给系统面子，是在训练你自己的工具。

## 三、判断标准（系统是怎么想的）

系统按博士的判断法工作，你知道这几条就能理解它的脾气：

- **先筛后析**：每条热点先过"值不值得用"——碰监管立法、诉讼、天灾人祸、贩卖失业焦虑、细节不足、太小众的，直接"别蹭"，不浪费分析。
- **不看热度看关系**：热度 9 分但跟你生意没关系 → 别蹭；热度 4 分但正中你客户的痛 → 必发。别被热度排序迷惑。
- **牵强即放弃**：需要绕好几个弯才能接上你产品的，系统宁可不推——硬蹭伤号。
- **争议可以做**：客观现实带来的大争议会标"拍板"+风险提醒，避免草稿过满或过激。

## 四、网站外怎么改账号

- **新增账号**：找项目维护者处理，不是审批。维护者会打开 `/onboarding` 填问卷，把生成的账号 JSON 保存到 `data/accounts/<account_id>.json`；如果是新赛道，也把赛道 JSON 草稿保存到 `config/tracks/<track_id>.json`。保存后必须跑一次 `status.py --preflight <account_id>`。
- **改账号信息**：直接改 `data/accounts/<account_id>.json` 的 `memory` 字段，下一次跑批生效。
- **改搜索母题**：编辑 `config/tracks/<track_id>.json` 里的赛道热点方向；这会影响该赛道所有账号。
- **今天没更新**：页面是昨天的日期，说明今天还没跑批。按 RUNBOOK 跑完 `make-prompt → ingest → status` 后刷新页面。

## 五、常见问题

**Q：今天我的账号一条必发都没有？**
正常。说明今天的热点没有能自然接住你生意的，硬发不如不发。可以看看"拍板"里有没有你愿意赌的角度。

**Q：我觉得某条"别蹭"其实能发？**
点开它看理由；还是不服就在打分的备注里写"这条我觉得能发，因为……"——这正是系统最需要的反馈。

**Q：脚本口吻不像我？**
打分时在"角度对吗"给低分+备注说明。连续几次后，把这些反馈整理进账号 JSON 的 `memory.content_style` 或禁区字段。

**Q：能看到以前的推荐吗？**
左侧"历史档案"按日期回看任何一天。
