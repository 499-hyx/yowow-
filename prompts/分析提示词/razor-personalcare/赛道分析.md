# razor-personalcare · 赛道分析层

用于 generate 阶段。`scripts/make-prompt.py` 会把本文件自动注入到 `generate-*.txt` 的 `{track_analysis}`；外部 LLM 只需按 `generate-*.txt` 规定的 JSON 输出。

## 第 0 步：先判值不值得做

命中任意一条，直接 `skip`：

1. 和男性个护 / 日常体面 / 出门效率没有自然关系。
2. 只能靠“男人也要精致”硬拐。
3. 需要医疗承诺、治疗承诺、皮肤病判断。
4. 需要攻击竞品或做极限低价比较。
5. 事件细节不足，无法支撑内容。

## 判断顺序

1. 热点里普通人真正关心什么？
2. 这件事能不能自然落到“男性可控的日常确定性”？
3. 产品价值能不能接上“出门前几分钟拿到体面感与效率”？
4. 抖音老板型账号能不能用直接、克制、不贩卖焦虑的口吻说清？

## 推荐表达

- 体面不是给别人演，是给自己一个确定状态。
- 清爽不是精致焦虑，是出门前可控的小事。
- 省时间不是偷懒，是让重要场合少出错。
- 产品好不好，不先看品牌，先看真实体验、参数和上脸效果。

## 生成结构

`content.body_or_script` 按这个内在结构写：

1. 热点事实：客观发生了什么。
2. 普通人为什么在意：它戳中了什么状态焦虑。
3. 男性日常体面关系：这件事怎么自然落到出门前、自我管理、清爽感。
4. 产品价值：一把顺手剃须刀能解决什么具体不确定性。
5. 风险自检：不做医疗承诺，不制造外貌焦虑。
6. 行动建议：今天就能检查或选择的一个具体动作。

## 必须遵守

- 非 skip 必须给至少 3 条 `bridge_paths`，每条 5 步齐全。
- 不出现禁词：远迁移、OOD、范式转移、供应链黑话、内部参数代号。
- 不写竞品品牌词，不写“根治/治愈/包好”。
- 牵强就 `skip`。
