# petfood-sourcing · 赛道分析层

用于 generate 阶段。`scripts/make-prompt.py` 会把本文件自动注入到 `generate-*.txt` 的 `{track_analysis}`；外部 LLM 只需按 `generate-*.txt` 规定的 JSON 输出。

## 第 0 步：先判值不值得做

命中任意一条，直接 `skip`：

1. 和养宠、口粮、配料、来源、信任没有自然关系。
2. 只能靠“宠物也一样”硬拐。
3. 需要医疗诊断或治疗承诺。
4. 需要编造检测报告、原料溯源或用户反馈。
5. 纯低价促销或攻击竞品。

## 判断顺序

1. 热点里养宠人真正担心什么？
2. 这件事能不能自然落到“喂得安心”？
3. 产品价值能不能接上“配料看得懂、来源查得到”？
4. 抖音老板型账号能不能用源头、实在、不夸大的口吻说清？

## 推荐表达

- 选粮别先看广告，先看配料表第一行。
- 养宠人不是想当专家，是怕花了钱还喂错。
- 来源说得清楚，是源头敢被看的底气。
- 大牌不等于适合你家猫狗，配料和反馈更重要。

## 生成结构

`content.body_or_script` 按这个内在结构写：

1. 热点事实：客观发生了什么。
2. 养宠人为什么在意：它戳中了什么喂养焦虑。
3. 配料 / 来源关系：这件事如何自然落到“看得懂、查得到”。
4. 产品价值：源头好粮能提供什么确定性。
5. 风险自检：不做医疗承诺，不编造检测或反馈。
6. 行动建议：今天买粮前能检查的一件具体事。

## 必须遵守

- 非 skip 必须给至少 3 条 `bridge_paths`，每条 5 步齐全。
- 不出现禁词：远迁移、far transfer、OOD、范式转移、包治百病、速效增肥、纯天然无添加、包治挑食、进口神话。
- 不编造检测报告、原料来源、真实饲喂反馈。
- 牵强就 `skip`。
