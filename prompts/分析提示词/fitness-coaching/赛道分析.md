# fitness-coaching · 赛道分析层

用于 generate 阶段。`scripts/make-prompt.py` 会把本文件自动注入到 `generate-*.txt` 的 `{track_analysis}`；外部 LLM 只需按 `generate-*.txt` 规定的 JSON 输出。

## 第 0 步：先判值不值得做

命中任意一条，直接 `skip`：

1. 和训练、身体改变、坚持、动作风险没有自然关系。
2. 只能靠“健身也一样”硬拐。
3. 需要医疗治疗承诺。
4. 需要几天速瘦、包瘦、减肥药、躺瘦等违规表达。
5. 以羞辱身材制造焦虑。

## 判断顺序

1. 热点里普通人真正卡在哪里？
2. 这件事能不能自然落到“科学训练 + 可坚持”？
3. 私教价值能不能接上“计划、纠错、陪练、体测数据”？
4. 内容能不能给出一个不夸张、可执行的小动作？

## 推荐表达

- 普通人缺的不是狠劲，是可执行的计划。
- 练错比不练更容易打击信心。
- 看得见的进步来自动作标准、负荷递进和稳定复盘。
- 不用整块时间也能练，但要知道练什么、怎么练。

## 生成结构

`content.body_or_script` 按这个内在结构写：

1. 热点事实：客观发生了什么。
2. 普通人为什么代入：练不动、练不对、坚持不了。
3. 训练方法关系：这件事如何自然落到科学训练和动作标准。
4. 产品价值：私教 / 训练计划能提供什么确定性。
5. 风险自检：不承诺速瘦，不制造身材羞辱。
6. 行动建议：今天能做的一个训练或判断动作。

## 必须遵守

- 非 skip 必须给至少 3 条 `bridge_paths`，每条 5 步齐全。
- 不出现禁词：远迁移、far transfer、OOD、范式转移、增肌神药、几天速瘦、包瘦、减肥药、躺瘦。
- 不写医疗建议，不做绝对效果承诺。
- 牵强就 `skip`。
