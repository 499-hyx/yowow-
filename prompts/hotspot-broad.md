# 任务：找出过去两周普通人在小红书、抖音等平台上真正自发热议的话题

今天是 {date}。请扫描**平台原生讨论**——小红书、抖音、微博、B站、知乎，以及国外的 TikTok、Reddit、X、Instagram——找出过去两周**普通人以第一人称在大量讨论、转发、争吵、跟风**的话题。你不知道这些将被用于什么行业——你的唯一任务是如实报告"普通人在聊什么"。

## 要什么、不要什么（核心口径）

**要的**是这样的话题：
- 普通人自己发帖、晒图、吐槽、对线、玩梗、跟风模仿的——话题的主角是用户，不是记者；
- 可以完全没有新闻由头：一种突然流行的生活方式、一个引发站队的生活争议、一个全网模仿的格式/挑战、一句突然刷屏的话；
- 新闻事件也可以收，但**只收已经长出平台原生讨论的部分**——大家在用自己的生活经验吵它、二创它，而不是单纯转发通稿。

**不要的**：
- 只有官方通报、媒体报道、热搜词条，但找不到普通人第一人称讨论痕迹的；
- 纯明星八卦、纯吃瓜、纯猎奇（除非它已经变成普通人讨论自己生活的由头）；
- 平台官方策划的营销话题。

## 扫描面（每个方向找 1-3 条，找不到就空着）

1. **生活方式风潮**：突然流行的吃法、用法、活法、消费方式（含"反向消费""平替"类）
2. **生活争议与站队**：让评论区吵起来的日常选择题（家庭、消费、习惯、边界感）
3. **育儿与家庭圈热帖**：家长圈自发刷屏的经验帖、焦虑帖、争议帖
4. **打工人话题**：职场吐槽、收入讨论、新活法（辞职、副业、躺平卷起的新变体）
5. **身体与状态**：普通人晒的健康实验、身体变化、习惯打卡、新式养生
6. **梗与格式**：全网模仿的句式、挑战、拍法、表情包语境（要能说清它为什么火）
7. **关系与情感**：婚恋、代际、友情、社交方式的热议帖
8. **海外平台正在火的**：TikTok/Reddit/X 上的热议话题或风潮，特别是可能传入中文平台的

## 入池门槛（每条必须全过）

1. **平台原生证据**：能给出话题 tag、代表性帖子/视频的说法、大致声量（多少赞/多少讨论/多少二创）；找不到 UGC 痕迹的不收。深度检索时可借助媒体对热搜/热帖的转述、数据榜单、tag 汇总页等间接证据，但必须指向真实的用户讨论。
2. **第一人称浓度**：大量内容是"我家/我司/我自己"的现身说法，不是转发新闻。
3. **跨人群**：至少两类不同的人在用不同立场聊它。
4. **有冲突结构**：说得清谁和谁、什么期待和什么现实在拧巴。
5. **能读出 2 种以上**不同的焦虑或欲望；时间窗两周，更早的除非正在翻红不收。

总量 10-20 条，热度按平台真实声量粗估（0-10），不做任何行业取舍。

## 加量跑法：全景 + 专场（一天可多跑几轮）

单次深检索受工具限制，产量上限约 12-15 条。要更大的池子，不放宽门槛，而是**多开专场**：

- **全景场**（1 次）：照上面 8 个扫描面跑一遍。
- **专场**（每天挑 2-4 个轮换）：把「扫描面」整段换成**单一方向**，配额提到 5-8 条，并指定主攻平台。例：
  - 「只扫【育儿与家庭圈热帖】，主攻小红书和抖音，找 5-8 条」
  - 「只扫【海外平台正在火的】，主攻 TikTok 和 Reddit，找 5-8 条」
  - 「只扫【生活争议与站队】，主攻微博和抖音评论区，找 5-8 条」
- 各场结果**合并去重后**一起过 neutralize 入池；同一话题以证据更全的一条为准。
- 专场平台每天轮换（今天小红书专场、明天 B站/知乎专场），避免池子长期偏向单一平台生态。

## 输出格式（严格输出下面这个 JSON 数组，不要多余文字）

```json
[
  {
    "title": "<话题，用普通人自己的说法命名，不用新闻标题腔>",
    "summary": "<2-3 句：大家在干什么/吵什么、什么人在参与、为什么突然火>",
    "happened_when": "<大致时间，如 2026-06 上旬>",
    "platform_evidence": "<哪个平台、什么 tag/代表帖、大致声量；国外话题注明平台和是否已传入中文圈>",
    "ugc_voice": "<一两句普通人原话风格的典型说法（评论区/帖子里的腔调）>",
    "scan_domain": "<命中上面哪个扫描面>",
    "conflict_hint": "<一句话：谁和谁、什么和什么在拧巴>",
    "problem_dimensions_hint": ["<它戳中的焦虑/欲望，至少 2 条>"],
    "est_heat_score_10": 0.0
  }
]
```

## 护栏

- 全程不出现任何行业、品类、产品视角；不猜"这条对谁有用"。
- 找不够 10 条就如实输出实际条数，绝不编造话题、不虚构声量。
- 本步只负责扫描，中立化富化交给 neutralize.md。
