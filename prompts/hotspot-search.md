# 任务：为「{track}」赛道定向召回热点候选

今天是 {date}。{search_brief}：

{directions}

<!-- <<< 可编辑区 开始 ====================================== 博士在此调召回口径 -->

## 召回口径（在此编辑）

1. **只要「自然引出」，不要「硬蹭得上」。** 一条新闻能召回，是因为普通人看完会自然
   联想到上面某条话题方向；需要你先解释半天才能扯上的，不算召回。
2. **时间窗：过去一个月内**有真实讨论度的新闻、事件、研究发布、平台热议；
   常青旧闻除非最近重新被点燃，否则不收。
3. **真实优先**：必须是真实发生、可给出来源线索的事件。记不清出处、疑似编造的不收。
4. **每条方向召回 0-3 条即可**，召回不到就空着，不为凑数降低标准。
5. 同一事件命中多条方向时只出一次，标注命中的全部方向。

## 输出格式（严格输出下面这个 JSON 数组，不要多余文字）

```json
[
  {
    "title": "<事件标题>",
    "summary": "<2-3 句：发生了什么、谁在讨论、为什么有热度>",
    "happened_when": "<大致时间，如 2026-05 下旬>",
    "source_hint": "<来源线索：媒体名/平台名，可给 URL>",
    "hit_directions": ["<命中的话题方向原文>"],
    "why_natural": "<一句话：普通人为什么看完会自然想到这个方向>",
    "est_heat_score_10": 0.0
  }
]
```

## 护栏

- 召回为空就输出 `[]`，绝不编造事件。
- `why_natural` 写不顺的条目，删掉。
- 本步只负责召回，不做赛道归属判断、不写任何内容方案。
