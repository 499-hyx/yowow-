# 任务：把原始热点中立化

今天是 {date}。下面是抓取层抓到的一条原始热点候选，请把它富化成一条**中立热点**。

## 原始候选（fetch-v1 原料）

标题：{hotspot_title}

```json
{hotspot_raw}
```

<!-- <<< 可编辑区 开始 ====================================== 博士 / 老板调口径 -->

## 中立化口径（在此编辑）

> 改这里 = 改「怎么把一条热点拆成中立原料」的标准。引擎不碰这段。

1. **只读出、不搬运。** 不要把原料里的 summary / why_rising 整段抄过来当 phenomenon。
   要用你自己的话，把「客观发生了什么」归纳成一句干净的现象。

2. **中立 = 不预设赛道。** 现象、情绪、问题维度都不要替任何赛道说话。
   例如别写「这说明孩子需要 XX 能力」——那是适配层的事。这里只把原料抓厚、保持中立。

3. **candidate_problem_dimensions（候选问题维度）= 这个现象可能戳中哪些类型的真实问题**，
   多写几个、各自独立、不绑赛道。后面适配层会据此判断「能不能接到某条赛道」。

4. **缺料就老实说缺料。** 如果原料太薄，读不出客观现象 / 判不出传播情绪 / 抽不出问题维度，
   **不要硬编**——把 `needs_human` 设为 true 并写明缺什么，交人工补料。

<!-- <<< 可编辑区 结束 ============================================================ -->

## 输出格式（严格输出下面这个 JSON，不要多余文字）

```json
{
  "needs_human": false,
  "needs_human_reasons": [],
  "neutral": {
    "title": "{hotspot_title}",
    "summary": "<可选：一两句客观概述>",
    "phenomenon": "<现象：客观发生了什么，用你自己的话归纳>",
    "spread_emotion": "<传播情绪：大家在激动 / 焦虑 / 嘲讽什么>",
    "people_involved": ["<涉及人群>"],
    "conflict_point": "<可选：冲突点 / 反差点>",
    "fact_structure": "<可选：因果 / 对立 / 数字骨架>",
    "candidate_problem_dimensions": ["<可能戳中的真实问题1>", "<问题2>", "..."]
  }
}
```

若判定缺料，则：

```json
{ "needs_human": true, "needs_human_reasons": ["<缺什么，例如：无 key_facts 也无 summary，现象无从落地>"], "neutral": null }
```

## 护栏

- 三个必填字段 `phenomenon` / `spread_emotion` / `candidate_problem_dimensions` 必须是**你的归纳**，
  不是原料字段的整段照抄；`candidate_problem_dimensions` 至少 1 个、建议 ≥2 个。
- 这些中立字段里**不得**出现任何内部黑话（远迁移 / far transfer / OOD / 范式转移 等）。
- 拿不准就 `needs_human=true`，宁缺毋造。
