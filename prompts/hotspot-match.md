## 匹配判断标准（在此编辑）

> 改这里 = 改「怎么算接得住、怎么算牵强」。引擎不碰这段。

判断分两个维度，各打 0-10 分：

- **相关度（精准流量信号）**：这条热点戳中的真实问题，和本赛道客户的焦虑
  （`{anxiety_anchors}`）契合得有多紧？契合越紧、越是「接得住本赛道」的精准流量。
  **注意：热点本身的流量大小（谁都在看）不算相关度**，别被泛流量带偏。

- **自然度**：用后台理解锚 `{internal_lens}` 判断——这条热点能不能**自然**承载本赛道主张？
  桥是自己长出来的（自然度高），还是要绕很远 / 硬掰才搭得上（自然度低）？

定档规则：

- 自然度 < 5，或者怎么想都得硬掰 → **skip（不建议蹭）**。宁可不推，也不生蹭流量伤号。
- 相关度高 且 自然度高 → **strong_pick（高匹配）**。
- 介于两者之间、能接但角度有点绕 → **maybe（可尝试）**。

（2026-06-10 老板批注，最终验收标准）：

- 这条推荐成不成立，看的不是「能不能写出一篇内容」，而是**老板看完会不会觉得：
  「这个热点我原来不会接，但它帮我接得很自然」**。需要强行跳跃才能接上的，一律 skip。
- 只是因为热点流量大、但和产品价值关系弱的，标 **skip（不建议蹭）**，理由里直说「流量大但跟你的生意关系弱」。
- 只有「热点背后的真实问题 → 目标用户焦虑 → 产品价值 → 平台表达」这几步都顺，才进 strong_pick / maybe。
- 如果赛道配置里带 `bridge.search_directions`（博士给的账号级桥梁方向），优先判断这条热点
  能否**自然引出**其中某个话题方向；能自然引出的优先抬档，引不出的不因此扣分。

## 输出格式（严格输出下面这个 JSON，不要多余文字）

```json
{
  "tier": "strong_pick | maybe | skip",
  "relevance_score": 0.0,
  "naturalness_score": 0.0,
  "why_relevant": "<一句人话：这条热点戳中的真实焦虑，跟本赛道怎么对上的（给老板看，零内部术语）>",
  "skip_reason": "<仅当 tier=skip：一句人话说明为什么不建议蹭>"
}
```

## 护栏

- `why_relevant` 和 `skip_reason` 是**给老板看的人话**，**绝不能**出现内部术语
  （`{internal_lens}` 里的词、或 `{forbidden_terms}`、或 relevance / naturalness 等分数名）。
- 牵强即 `skip`，不要为了多推一条而把 maybe 往上抬。
- `internal_lens` 只用来在你脑子里判断自然度，**判断完不要把它写进任何输出字段**。
